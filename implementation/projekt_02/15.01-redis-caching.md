# Arbeitspaket 15.01: Redis Caching Layer

## Ziel
Implementierung einer Redis-basierten Caching-Schicht mit Upstash Redis für API-Responses und häufig abgefragte Daten.

## Kontext
- **Service**: Upstash Redis (Serverless Redis)
- **Library**: @upstash/redis
- **Serialization**: superjson für komplexe Datentypen
- **Pattern**: Cache-aside mit TTL und Tag-based Invalidation

## Implementierungsschritte

1. **Upstash Redis Setup**
   - Account bei Upstash erstellen
   - Redis Database erstellen
   - Environment Variables konfigurieren

2. **Cache Manager Implementation**
   - `packages/api/src/cache/index.ts` erstellen
   - CacheManager Klasse mit get/set/delete/invalidate
   - Tag-based Invalidation Support
   - Automatisches Serialization/Deserialization

3. **Cacheable Decorator**
   - Higher-Order Function für automatisches Caching
   - Key Generation aus Funktionsparametern
   - TTL und Tag Configuration

4. **Integration in API Layer**
   - Cache Integration in oRPC Procedures
   - Invalidation bei Mutations
   - Cache Warming Strategies

## Erfolgskriterien
- [ ] Upstash Redis konfiguriert
- [ ] CacheManager implementiert
- [ ] Tag-based Invalidation funktioniert
- [ ] Cacheable Decorator einsatzbereit
- [ ] API Integration getestet
